Intro to Reinforcement Learning

Hello Everyone 😀 Welcome to this first post ! Today we are going to talk about Reinforcement Learning (RL). This post is a beginner friendly introduction to RL. I will explain what is RL, why RL, what are some of the most important challenges in the fields of RL and some common applications of RL.

Introduction


Animals, including humans, learn by interacting with their environments and they do so continuously. For instance, when an infant plays, waves its arms or looks about, he receives tons of information about the world thanks to his senses. He learns about the causality i.e. causes and consequences of his actions. He tries things and learns about himself and the environment from that experience. In order to learn how to solve a new task, he must do lot of trial-and-errors depending on the complexity and difficulty of the task at hands, without necessarily relying on any teacher.

The purpose of Reinforcement Learning is to provide computational methods to learn from experience. RL is thus learning what to do, how to map situations to actions, by maximizing a numerical reward signal. The promise of RL is to allow machines to learn as any other animal by interaction with their environment and continuously. An RL agent interacting within an environment does not know the causality in that environment i.e. it is not told which actions yield the most reward but must discover that by trying them. Some of the most distinguishing characteristic of RL is that the actions of the RL agent may affect not only the immediate reward but also the next situations and, through that, all subsequent rewards. The term "situations" could be replaced by "states" or "observations". These three terms are similar in practice and represent whatever information that are useful to a learning agent to solve a given task. For example, in a game of chess the state is the chess board with all current pieces on it. In a robotic navigation, the state may be the robot velocity, position, orientation, etc. In fact, each of these quantities can constitute a state itself, but also their combination can constitute a unique state.

Note that, for this late example, the state of the world or the environment is not fully known unlike in the game of chess because we have decided to select some aspects of the world and by selecting those we discarded others, like visual information. More on this in future section.


How Reinforcement learning differs from supervised learning and unsupervised learning?


Machine Learning is generally classified in 3 paradigms: Supervised Learning (SL), Unsupervised Learning (UL) and Reinforcement Learning (RL).

In Supervised Learning (SL)the learner receives a bunch of examples called training data provided by an external expert from which it learns how to behave correctly. The data are provided as pairs of inputs and their corresponding target/label. During inference, the learner is provided with the input and it must predict the correct output even for the inputs that it did not see during its training phase. That is to generalize to unseen data. SL is generally used in regression tasks (e.g. predict the housing or stock price, etc) and classification tasks (e.g. classify an image as dog, cat, humans, etc). This type of learning, although very useful, is not well suited to learn from interaction alone since it requires an expert to tell the learner which situations to map to which actions.

On the other hand, Unsupervised Learning (UL) is about finding structure hidden in unlabeled data. Even though UL and RL don't require examples of correct behavior during their training phase, they are different because RL is about solving a task by maximizing a reward signal instead of finding hidden structure. Nevertheless, UL methods are frequently used in RL to learn better representations of the world in order to solve a task.


What problem does it try to solve ?


RL methods try to solve goal-directed sequential decision making problems i.e. in order to achieve its goal, a learning agent must find a sequence of actions to reach that goal.

RL problems are typically formulated as a Markov Decision Process (MDP) where an agent interacts with an environment at discrete time steps. The agent is learner, the decision maker and the environment is everything outside of the agent.

MDP is a mathematical framework defined by the tuple {S, A, T, R} where S, A, T, R represents respectively the state space, action space, transition matrix and reward function. The state space represents the set of possible situations that the agent may see by interacting with the environment; action space is the set of possible actions that the agent can take; the transition matrix is a a distribution that determines the probability of transitioning from a source state to a destination/next state by taking an action, it's usually represented by T(s, a, s'); and finally the reward function R(s,a) represents the reward received by taking action a in state s. Below is a typical illustration of the interaction between an agent and its environment in a MDP setup.

During an interaction with its environment, at each time step, the agent receives some representation of the environment also called state or observation. Given that state, it can take actions and gets feedback from the environment in the form of a reward signal that informs the agent about the goodness (or badness) of its actions and then ends up in next state. Its sole goal is to solve the task by maximizing the cumulative reward signal: this is called the Reward hypothesis.

This reward signal is designed by the user and therefore must be designed in such a way that maximizing it will achieve the user intended goal. For example, to learn how to play the game of chess, the reward may be +1 in case of a win, -1 for a lost and 0 for a draw; to learn a locomotion task (walking for a humanoid or quadruped robot), the agent may receive a reward proportional to its forward speed, etc.

To maximize its cumulative reward, the learning agent must learn how to win ! Easy, isn't !? Of course, not all tasks can be solved by maximizing a binary reward as in the game of chess. For example, in robotic navigation, reaching goal position may be the main goal, but other goals such as minimizing energy usage, maintaining a target orientation, etc require sophisticated reward design. This process of designing a good reward function to solve your task is called Reward Engineering or Reward Shaping. This part is very likely one of the most time consuming and most important when it comes to apply a Reinforcement Learning algorithm.


How does an RL agent learns to solve a task ?


In the beginning of its learning phase, an RL agent possesses little or no knowledge about how its actions affect its environment i.e. it does not know whether an action will yield a positive or negative reward signal and which situations an action or a sequence of action will lead to. Therefore, it must try them ! By long trial-and-error, it will learn to take actions that lead to good situations i.e. good reward signal whilst avoiding the ones that lead to bad situations. At the end of the training, an RL agents has learned what is known as an optimal policy. An optimal policy is a policy, i.e. simply a function, that maps the best action to each situations. The best action, is of course the one leading to the highest cumulative reward signal. During this series, we will see how this is done in practice, that is, how it is formulated mathematically and how it is implemented programmatically and deployed on a real or virtual learning agent to solve a task.

Examples of problems solved with RL?


As previously mentioned, RL algorithms try to solve sequential decision making problem and there are lot of applications to such methods where they may be used to solve tasks that real human actors can solve. Sometimes even better than human actors.

The most famous field of applications for RL algorithms is Video Games (VG). In Video Games, the player must take sequences of actions to reach a goal and get rewarded or punished depending on the outcome of the game. The situation or state or observation in such case may be video frames of the game, or information related to the characters (player or enemies) in the game that are relevant for wining.

The game of "Go" was reputed to be hard to solve for machine because of the incredible number of possibilities that may raise during a game play, in the order of 10²⁵⁰. Far greater than the number of atoms in the observable universe. In the game of Go, the state would be the board (represented as a matrix) with all stones (pieces of the game of Go) played up-to-now and the reward signal would be as the game of chess (+1 for a win, -1 for a lost, 0 for a draw).

In 2016, DeepMind invented AlphaGo to beat the top human player at the time, Lee Sedeol (I recommend watching AlphaGo the Movie on YouTube, a master piece !). AlphaGo was based mainly on RL methods, specifically Monte Carlo Tree Search (MCTS) algorithm. A more advanced version of AlphaGo is AlphaZero which learned the game without any human prior (except for the reward design phase of course) by simply playing against itself with RL. By playing against itself (self-play), and by trial-and-error, AlphaGo learned all the strategies and tactics to win a game of Go simply with a scalar reward (Is scalar reward enough to develop all necessary intelligence to solve Go?) No human can beat the best machines today !

Another use case of RL in real world is self-driving cars. In fact, any mobile robot can be controlled with RL. Mobile robots navigation require control and planning. Both of these can be modeled as a MDP thus solvable with RL algorithms. The observation may be vehicle odometry (speed, orientation, etc), goal information, energy state of the vehicle, Lidar point clouds, videos frame if the robot has a camera, GPS, maps, etc. A good reward function will necessary include penalty for hitting obstacles, penalty for driving faster than the authorized speed limit, reward for reaching the destination and more. In fact, one may train a RL agent to achieve anything a human driver could achieve with a car such as overtaking, lane changing, etc, the limit is in the design of the reward function. For example, what reward function to respect ethical and social aspects of self-driving cars ? or for a motorbike for example do cascades with the motor bike ? you see that things can be trickier !

One of the most recent and most powerful use case of RL is in the fine-tuning process of Large Language Model (LLM) into chat bot (chatgpt, deepseek-R1, gemini, etc). Pre-trained LLMs are known to be good at predicting the next word (or token) in a sentence after being trained on the whole internet. They write well structured and grammatically correct sentences. However, they often have some issues such as hallucination, misinformation, generating harmful and sensitive contents, etc. In one sentence, they are not yet aligned with human values. This is where enters RL, mostly Reinforcement Learning from Human Feedback (RLHF) ! By using RLHF, the LLM model learns to generate responses that are aligned with human preferences, responses that are helpful, harmless, concise, creative, etc. RL allows to have more control over LLMs behavior and helps mitigate negative behaviors through the reward function.

The agent is obviously the LLM model itself, the state or observation is the user prompt(s) and the system prompt or description of tools available in case of agentic LLM. The action could be choosing between different potential answer to a question in order to maximize the reward which could be about how helpful or harmless or concise, the answer is. Actions could also be whether to perform reasoning to answer a given question or not. The reward is usually designed to tell the LLM how good or bad is the generated output.

There exist so many other applications of RL that I didn't mention here such as industry automation, financial trading, task scheduling, etc.


Challenges in RL:


I will talk about some of the many challenges that ML Engineers and Researchers are facing in Reinforcement Learning. If you're an engineer who only focus on applications of RL, you may not face some of the challenges that I will mention but they are to be overcome those researchers willing to develop new RL algorithms.

When you're an engineer, the first step to do when you want to use a given RL algorithm to solve your task is to design your environment in order to train your model. Because RL methods are reputed to be sample inefficient, they often require lot of interactions (from thousands to millions of actions taken to learn how to solve a task depending on the complexity or difficulty of the task at hand). Therefore, you cannot afford to train in real environment thus needs to build a realistic environment to train your model in before deploying it in real world. This part is very tricky because if your simulated environment reflects poorly the real world (because you made it too perfect or forget about some important aspects of environment), the policy that your agent will optimize within simulation will inevitably under-perform in real life. For example in autonomous navigation, the dynamics of the environment must reflect the world dynamics in which the robot will be deployed

If you want your robot to be robust to strong disturbances, then you must apply strong random disturbances during training so that your model learns to generalize to those situations. Likewise, a trading agent must be trained in realistic environment with fluctuations that mimics those of real people so that your model can develop strategies that have a chance in real life. This seems obvious but it's very important to keep it in mind and understand that RL does not do miracles. Don't be surprised if you see your agent performing poorly in situations you didn't show it during its training phase.

Another challenge that you will face when you plan to use RL is of course reward engineering. That is to say, designing a reward function or model that acts as a judge and will give feedback to the agent whenever it takes an action in a given situations. It needs to be designed in such a way that maximizing it will solve your task. You may choose to give a reward signal only at the end of a trial, give positive reward when the goal is reached and negative reward when the goal is not reached. This is likely the cleanest way to let the agent express itself and find out all the necessary skills required to maximize that reward signal. For example, in the game of Go, to maximize its wining chance to receive + 1 reward and avoid the cost of losing, AlphaZero learned about attack and defense, discovered new opening sequences, learned to play safe when ahead, etc.

However, this reward design technique has some drawbacks: not all tasks have a clear win or lost position, and it's slow to converge because the agent must first reach the goal or lose to even have any meaningful feedback which give raise to the credit assignment problem (more on this later). To illustrate the first, let's consider an autonomous navigation task like reach a target position. In such case, giving a binary reward (0 until goal and +1 when the target is reached) will mean that the only objective here is reaching the goal, which is not always true. The agent might accelerate all his way to the goal, or doing some zig-zag, or going backward, etc. We also want to use less actuator energy, maximize comfort (no strange movement like a zig-zag, etc). To accelerate the learning phase, we may give reward feedback in a continuous way in order to guide the agent toward the target position quickly. Hence the reward function is usually a continuous function that the agent receives at each time step. It may be composed of a distance based term (e.g. negative distance from the target position, therefore any action that reduces the distance to the target is usually considered good), an energy term to penalize excessive use of actuators, a safety term (e.g. negative reward when the agent gets too close to an obstacle) and even some additional terms for comfort (e.g. negative reward for frequent change of vehicle orientation or its acceleration).

A challenge that inevitably arises in all RL task, and that is not usually faced in other type of machine learning methods like SL and USL, is the exploration-exploitation trade-off. As mentioned previously, an RL agent training phase consists of mapping the situations to the best actions possible. During that learning, it needs to explore its action space by trying different actions in different states. However, during its interaction within the environment, the RL agent might discover that one or several actions yield good rewards. One of two things here, it can either continue selecting that action that it has had good experience with, or try other actions that it has not tried before as we saw in the above example (in state C). Because, what if there exist a better action that has not been tried yet ?

In fact, this is a dilemma that our RL agent is currently facing here, therefore it needs to find a trade-off between always choosing the action that yields the most reward, that is called Exploitation, or sometimes choosing another action that it didn't experience before, that is Exploration. This exploration is really important mostly in stochastic or non-stationary tasks where the agent needs to try each actions many times in order to have a reliable estimates of their expected reward.

This exploration-exploitation trade-off problem has been studied for a long time by the RL researchers around the world and still remain unsolved (mathematically) though some heuristics exist that have proved good results in practice.

Now let's talk about one the most important challenge in RL, the credit assignment problem (CAP). CAP is about evaluating the long term influence of individual actions taken by an RL agent over the return. In order to successfully deploy RL agents in real life, it is important to solve the CAP since most feedback in real life can be noisy and delayed which makes it difficult to associate individual actions to their long term outcomes. To measure action influence, many RL methods use the so-called state-action value function (more on this subsequent posts in this series) where actions contribute as much as the expected return they achieve at termination of the episode.

There remains many other challenges that I didn't mention above which are still very important in the field of RL and some of them is sample inefficiency (RL methods often require lot of interaction to solve some tasks that may require just few iterations with a human) which can lead to huge computational cost hence a high environmental impact.

Some more open questions in the research are transfer learning which tackle the question of transferring or reusing the knowledge of the agent in one task to another task that shares some similarity with the previous one; continual learning which studies agents that learn and update their behavior in an environment according to new incoming information in a continuous way; etc.


What are its advantages over other learning method?


One of the challenges that arise in reinforcement learning, and not in other kinds of learning, is the trade-off between exploration and exploitation. To obtain a lot of reward, a reinforcement learning agent must prefer actions that it has tried in the past and found to be effective in producing reward. But to discover such actions, it needs to try actions that it has not selected before. The agent has to exploit what it has already experienced in order to obtain reward, but it also has to explore in order to make better action selections in the future. The dilemma is that neither exploration nor exploitation can be pursued exclusively without failing at the task. The agent must try different actions and progressively select those that yield the most rewards. When the environment is stochastic, each action must be tried many times to have a reliable estimate of its expected reward. The exploration-exploitation dilemma has been studied by mathematicians for many decades, and still remains unresolved. Contrary to RL, the entire issue of balancing exploration and exploitation does not even arise in supervised and unsupervised learning.


Conclusion


In this chapter, we introduced what Reinforcement Learning is and how it differs from other well known form of Machine Learning such as Supervised Learning and Unsupervised Learning.

RL is a Machine Learning technique to tackle sequential decision problem by trial-and-error. In RL, the interaction of the agent within its environment is modeled as Markov Decision Process (MDP). MDP is a mathematical framework defined by the tuple {S, A, T, R} where S, A, T, R represents respectively the state space, action space, transition matrix and reward function. An RL agent has no knowledge of the causality in its environment in the beginning of its learning phase, thus must learn by trial-and-error. It learns by estimating long term consequences of actions to make a well informed decision.

We saw that RL can be applied to any sequential decision making problem like Video Games, Finance, Robotics, Chatbot, task scheduling, traffic routing, etc, basically any task that requires planning and decision making to achieve a goal.
